# Submission Checklist (PDF + GitHub)

## Required Deliverables
- Public GitHub repository URL (open access).
- Repository contains full source + README.
- Final PDF (max 4 pages) using the provided template.

## Repository Requirements
- Include all project code and assets.
- Include setup and execution steps in README.
- Include evaluation/testing notes and outputs.

## PDF Assembly Steps
1. Open docs/submission/4-page-summary-template.md.
2. Fill participant details and update all placeholder values.
3. Export to PDF (VS Code Markdown preview print, Typora export, or Pandoc).
4. Ensure max 4 pages and include public GitHub URL.

## Recommended Attachments for Stronger Scoring
- Architecture diagram (from README).
- Evaluation metrics from docs/evaluation/latest-results.json.
- Error analysis and mitigation notes.
- Demo screenshots/GIF links.

## Final Validation
- [ ] Repo is public.
- [ ] README includes environment setup + run steps.
- [ ] PDF includes GitHub link.
- [ ] PDF <= 4 pages.
- [ ] Metrics and testing evidence included.
